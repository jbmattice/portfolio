Scalable Accessible Curriculum Timeline System

Data visualization • System design • WCAG compliance

Designed and implemented a reusable, accessible timeline system across 100+ curriculum pages (by grade level and subject area) to visualize learning progression by quarter and week.

Challenges

  • Complex curriculum data across 100+ pages
  • CMS constraints and conflicting CSS
  • Need for screen reader and translation support (100+ languages)
  • No existing mobile-friendly solution

Solutions

  • Created reusable HTML/CSS pattern for scalability
  • Avoided image-based charts for accessibility
  • Ensured keyboard navigation and accessible links
  • Developed custom mobile CSS transforming charts into lists

Outcome

Delivered a scalable, accessible system used across 100+ pages, improving usability, accessibility, and consistency.

Accessibility Audit and Remediation Guidance

WCAG audit • Developer collaboration • Remediation guidance

Conducted a comprehensive WCAG 2.2 AA audit of a dynamic web platform, evaluating 55 success criteria and identifying critical accessibility issues.

Key Findings

  • Missing alt text and incorrect image handling
  • Color contrast and focus indicator issues
  • Layout and reflow problems
  • Unclear link purpose and labeling issues
  • ARIA and semantic HTML inconsistencies

Approach

  • DHS Trusted Tester methodology
  • Manual testing with NVDA and keyboard navigation
  • Tools including ANDI and IBM Accessibility Checker

Outcome

Provided detailed remediation guidance and collaborated with developers to improve accessibility and awareness across the platform.

Accessible Document Remediation (PDF/UA Compliance)

PDF/UA • Section 508 • Manual tag tree auditing

Remediated a complex, multi-table PDF document to meet Section 508 and PDF/UA accessibility standards, ensuring it was usable with screen readers and other assistive technologies.

Challenge

  • Complex multi-table layouts with nested structure
  • Correct table header scope and associations
  • Logical reading order across the document
  • Distinguishing meaningful content from decorative artifacts

Approach

  • Performed manual tag tree audits to verify document structure
  • Corrected Table Header Scope to support accurate navigation
  • Reviewed and adjusted reading order for screen reader usability
  • Managed artifact tagging for decorative elements
  • Validated the document against Section 508 and PDF/UA standards

Outcome

Delivered a remediated document with improved accessibility for screen reader users, more reliable table navigation, and clearer structural relationships for assistive technology.
